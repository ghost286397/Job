Модуль мультиязычности версия 1.0

Используемые классы: 
Translator, TransScript

Используемые ресурсы: 
Assets/Languages/... Файлы переводов + файл ключей

Изменения в шаблоне:
К префабу LABEL добавлен скрипт TransScript
Теперь скрипт меню отвечает за подгрузку всех статических данных, в том числе и переводов

Суть:
В скрипте меню разработчик выбирает интересующий его язык перевода или MULTI
с этой информацией при старте сцены инициализируется Translator и, в зависимости от выбранного языка или языка по умолчанию для устройства (в случае MULTI) создает словарь переводов <ключ, перевод>. Ключ берется из i-ой позиции ключа, перевод из i-ой позиции файла перевода.
Для того, чтобы каждый раз при присвоении текста не обращаться к Translator добавлен класс TransScript. Теперь текст, нуждающийся в переводе можно присвоить полю Text этого класса, после чего он сам его переведет и передаст текстмешу. Так же он пытается перевести текст своего текстмеша при старте.

Неприятные моменты:
 - Кол-во строк и их порядок во всех файлах переводов должны быть одинаковы. Несмотря на то, что это входит в порядок вещей контент отдела - здесь возможны затяжные танцы с бубном
 - Одновременно доступен только один язык.
 - Названия файлов переводов должны соответствовать их названиям в enum Languages

Положительные моменты:
 - Больше никакого еб$^&*@ .csv!
 - контент отдел поставляет переводы именно в таком формате. Это должно сократить время реализации мультиязычности ~ на 2 часа.
